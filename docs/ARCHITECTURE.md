# Architecture

## 1. 项目定位

这是一个**个人长期维护的和弦谱静态网站**，目标不是做公共平台，而是做一个：

- 可持续维护
- 数据格式稳定
- 页面阅读友好
- 支持 transpose
- 可托管到 GitHub Pages

## 2. 总体架构

### 2.1 内容层
- 每首歌一份独立文本文件
- 当前存放于 `src/data/*.md`
- 文件头使用 frontmatter 风格保存元数据
- 正文使用 `(Chord)` 内嵌歌词格式
- 当前版本采用自定义 loader 读取歌曲文件，而不是 Astro content collection

### 2.2 解析层
- 扫描正文文本中的 `( ... )`
- 将括号内容识别为 chord token
- 将 chord token 绑定到其后歌词位置
- 输出为可渲染的数据结构

### 2.3 转调层
- 只处理和弦，不处理歌词
- 支持：
  - 大小调
  - 升降号
  - slash chord
  - 常见后缀（m, maj7, sus4, dim, aug, 7, 9 等）
- 显示策略可配置为：
  - sharp 优先
  - flat 优先

### 2.4 展示层
- 首页：歌曲列表
- 单曲页：
  - 元数据
  - transpose 控件
  - 和弦谱渲染区
- 未来可扩展：
  - 搜索
  - 标签筛选
  - 收藏
  - 打印

## 3. 技术选型

- 框架：Astro
- 语言：TypeScript
- 部署：GitHub Pages
- 数据：仓库内静态文件

## 4. 设计原则

1. **源格式简单**：优先兼容现有 polygon 风格输入
2. **显示逻辑可替换**：解析与渲染分层
3. **转调逻辑独立**：便于测试和后续增强
4. **静态优先**：减少后端复杂度
5. **文档先行**：所有关键实现步骤有据可查
