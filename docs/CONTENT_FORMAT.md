# Content Format

## 1. 目标

采用一个**足够简单、便于手工录入、兼容你已有习惯**的格式。

核心约定：

- 和弦写作 `(Chord)`
- 和弦出现在其所对应歌词之前
- 网站负责自动把和弦显示到歌词上方

## 2. 文件格式

每首歌一个 `.md` 文件，当前放在 `src/data/`。

示例：

```md
---
title: 海阔天空
artist: Beyond
language: yue
originalKey: C
capo: 0
tags:
  - 粤语
  - 流行
  - Beyond
---

( C )今天我
( G )寒夜里看雪飘过
( Am )怀着冷却了的心窝
( F )飘远方
```

## 3. Frontmatter 字段

### 必填
- `title`: 歌名
- `artist`: 歌手
- `originalKey`: 原调

### 推荐
- `language`: 语言，如 `yue` / `zh` / `en`
- `capo`: 变调夹品位
- `tags`: 标签数组

## 4. 正文规则

### 4.1 基本规则
- `(Chord)` 视为一个和弦标记
- 和弦后的文本视为歌词内容
- 空行保留

### 4.2 示例

```text
( F )拦路雨偏似 ( C/E )雪花
( Dm7 )饮泣的你冻吗 ( C )
```

### 4.3 支持的和弦形态
- `C`
- `Am`
- `F#maj7`
- `Bb`
- `G/B`
- `Dsus4`
- `Cdim`

## 5. 非目标

当前版本暂不优先支持：
- 重型 lead sheet 指令系统
- 多声部/多轨标记
- 每小节拍号级别编辑
- 在线富文本编辑器

## 6. 兼容策略

第一阶段以兼容你既有 `(Chord)` 习惯为第一优先。
未来若要支持导入其他格式，应通过转换器实现，而不是破坏当前源格式。
